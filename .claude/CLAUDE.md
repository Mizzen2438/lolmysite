# Claude Codeへの基本指示

## 言語と話し方
- 日本語で回答する
- 専門用語は初出時に日本語で説明を添える
- わかりやすく、簡潔に伝える

## 作業前に必ず確認すること
- 3ステップ以上かかる作業は、実行前にやること一覧を提示して承認を得る
- ファイルを変更・削除する前に「何をするか・リスク・元に戻せるか」を説明する
- 既存ファイルの上書きは必ず確認してから行う

## Git規約
- Github Flow に則る
- **作業ブランチは毎回、最新の `origin/main` を基点に切る**。
  - 着手時に必ず `git fetch origin main` してから `git checkout -b <branch> origin/main` する。
  - セッションが既存ブランチ上で起動された場合でも、PR 用の変更は長命ブランチに積み増さず、
    最新 `origin/main` から新しいブランチを切って行う。
  - 理由: 古いブランチ上で進めると main との分岐点が古くなり、PR 差分にプロジェクト全体が
    出てしまう。基点を最新 main に保てば、PR 差分は当該作業だけになる。
- 変更を加える場合は必ず PR を作成し、ベースは `main` にする。

## やらないこと
- ファイルの削除（手順を提案して、実行は人間に任せる）
- 説明なしでコマンドを実行すること
- 承認なしで新しいパッケージをインストールすること

## 作業後
- 何を変更したかを箇条書きで要約する
